# 매수 스카우터 — 아이폰(iOS) App Store 출시 가이드

> Windows PC만으로 출시하는 구성입니다. **Mac이 없어도 됩니다.**
> 빌드·서명·업로드는 GitHub Actions의 macOS 러너가 대신하고,
> 서명 인증서도 클라우드 자동 서명이라 인증서 파일을 만들 필요가 없습니다.

## 이미 준비된 것 (Claude가 완료)

| 항목 | 위치 |
|---|---|
| iOS 네이티브 프로젝트 (Xcode) | `capacitor-app/ios/` |
| 앱 아이콘 1024px + 스플래시 (다크 포함) | `capacitor-app/assets/` → Xcode 에셋에 반영됨 |
| 버전 3.1 / iPhone 전용 / 세로 고정 | `ios/App/App.xcodeproj`, `Info.plist` |
| 수출 규정(암호화) 면제 선언 | `Info.plist` (`ITSAppUsesNonExemptEncryption=false`) — 심사 때 질문 안 나옴 |
| RevenueCat iOS 키 분기 (나중용) | `www/premium.js` (`RC_API_KEY_IOS`) |
| 빌드→TestFlight 업로드 자동화 | `.github/workflows/ios-testflight.yml` |
| 서명키·비밀번호 커밋 방지 | `capacitor-app/.gitignore` (`출시/`, `*.jks` 등 제외) |
| AdMob iOS 플러그인 + 광고 ID 분기 | `package.json`, `www/ads.js` (`*_IOS` 키), `Info.plist` (테스트 App ID) |

현재 앱은 `FREE_MODE: true`(구독 비활성) + **AdMob 광고(테스트 모드)** 구성입니다. 인앱결제가 없으므로 심사는 단순한 편입니다.
광고 수익을 받으려면 아래 **"AdMob에 iOS 앱 등록"** 단계까지 마쳐야 합니다 (TestFlight 확인까지는 테스트 광고로 충분).

## 1단계. Apple Developer Program 가입 — $99/년

1. 아이폰에서 사용하는 Apple ID로 https://developer.apple.com/programs/enroll/ 접속
2. **개인(Individual)** 으로 등록 (사업자 불필요, D-U-N-S 불필요)
3. 결제 후 승인까지 보통 몇 시간~2일
4. 승인되면 https://developer.apple.com/account > **Membership details** 에서
   **Team ID** (예: `A1B2C3D4E5`) 를 메모

## 2단계. App Store Connect API 키 발급

GitHub 러너가 애플에 로그인하는 데 쓰는 키입니다. 딱 한 번 만들면 됩니다.

1. https://appstoreconnect.apple.com > **사용자 및 액세스(Users and Access)** > **통합(Integrations)** 탭 > **App Store Connect API** > **팀 키(Team Keys)**
2. **(+) 키 생성** — 이름: `github-ci`, 액세스 권한: **App Manager**
3. 생성 직후 표시되는 것을 모두 메모:
   - **Issuer ID** (페이지 상단, 긴 UUID)
   - **Key ID** (예: `ABC123DEF4`)
   - **AuthKey_ABC123DEF4.p8 다운로드** — ⚠️ **한 번만 다운로드 가능**, 잘 보관 (절대 리포에 커밋 금지)

## 3단계. GitHub Secrets 등록

리포( https://github.com/qpt815-afk/stock-scores ) > **Settings** > **Secrets and variables** > **Actions** > **New repository secret** 로 4개 등록:

| Secret 이름 | 값 |
|---|---|
| `APPSTORE_ISSUER_ID` | 2단계의 Issuer ID |
| `APPSTORE_KEY_ID` | 2단계의 Key ID |
| `APPSTORE_P8` | `.p8` 파일을 메모장으로 열어 **내용 전체** 복사·붙여넣기 (`-----BEGIN PRIVATE KEY-----` 줄 포함) |
| `APPLE_TEAM_ID` | 1단계의 Team ID |

## 4단계. App Store Connect에 앱 만들기

1. https://developer.apple.com/account/resources/identifiers > **(+)** > **App IDs** > App
   - Bundle ID: **명시적(Explicit)** — `com.havacompany.maesuscouter` (안드로이드와 같은 ID라도 무관)
   - Capabilities: 기본값 그대로 → 등록
2. https://appstoreconnect.apple.com > **나의 앱** > **(+) 신규 앱**
   - 플랫폼: **iOS** / 이름: **매수 스카우터** (이미 사용 중이면 "매수 스카우터 – 주식 매수 타이밍 점수" 등)
   - 기본 언어: **한국어** / 번들 ID: 위에서 등록한 것 선택 / SKU: `maesuscouter-ios`

## 5단계. 코드 푸시

한글 경로(`앱만들기`)에서 node가 죽는 버그 때문에 로컬 빌드는 안 되지만, **푸시만 하면 CI가 빌드**합니다.

```
git add capacitor-app .github/workflows/ios-testflight.yml
git status   # ⚠️ capacitor-app/출시/ 가 목록에 없는지 반드시 확인 (서명키 유출 방지)
git commit -m "iOS 앱 추가 (Capacitor iOS + TestFlight CI)"
git push
```

## 6단계. 빌드 실행 → TestFlight

1. GitHub 리포 > **Actions** 탭 > 왼쪽 **iOS TestFlight** > **Run workflow**
2. 15~30분 후 초록색 ✓ 이면 업로드 완료
3. App Store Connect > 나의 앱 > 매수 스카우터 > **TestFlight** 탭 — 빌드가 "처리 중" 후 나타남 (5~30분)
4. 처음 한 번 "내부 테스트" 그룹 생성 → 본인 Apple ID를 테스터로 추가
5. 아이폰에 **TestFlight** 앱 설치(App Store에서) → 초대 메일 수락 → 매수 스카우터 설치·확인

> 실행할 때마다 빌드 번호가 자동 증가하므로(워크플로 실행 번호 사용), 웹(www/) 수정 후 다시 Run workflow만 누르면 새 빌드가 올라갑니다.

## 6.5단계. AdMob에 iOS 앱 등록 (광고 수익용 — TestFlight 확인 후 권장)

Android와 별개로 iOS 앱을 AdMob에 등록해야 실제 광고가 나옵니다.

1. https://apps.admob.com > 앱 > **앱 추가** > 플랫폼 **iOS** (아직 스토어 미출시 → "아니요" 선택)
2. 발급된 **앱 ID** (`ca-app-pub-XXXX~YYYY`, 물결표 포함)를
   `capacitor-app/ios/App/App/Info.plist` 의 `GADApplicationIdentifier` 값에 교체
   (지금은 구글 공식 테스트 App ID `ca-app-pub-3940256099942544~1458002511` 가 들어 있음)
3. 광고 단위 2개 생성: **배너**, **전면** → `www/ads.js` CONFIG 의
   `BANNER_ID_IOS`, `INTERSTITIAL_ID_IOS` 에 붙여넣기
4. 검토 통과 후 `ads.js` 의 `TEST_MODE: false` 로 변경 (⚠️ Android·iOS 공통 스위치이므로 Android 실제 ID도 준비된 상태에서 끌 것)
5. 커밋·푸시 → Run workflow → 새 빌드 제출

> AdMob 신규 앱은 광고 게재까지 검토가 며칠 걸릴 수 있습니다. 그동안은 광고가 비어 보여도 정상입니다.
> 앱스토어 출시 후 AdMob 콘솔에서 앱을 App Store 등록 정보와 연결해야 광고 제한이 풀립니다.

## 7단계. 스토어 등록 정보 작성

App Store Connect > 앱 > **배포(App Store)** 탭:

- **스크린샷 (6.9" 또는 6.5" 아이폰 1세트 필수)**: TestFlight로 설치한 앱을 아이폰에서 캡처하면 규격이 자동으로 맞습니다. 3~5장 권장 (`출시/스크린샷` 안드로이드 캡처는 해상도가 달라 재사용 불가)
- **문구**: `출시/스토어_문구_초안.md` 재활용 (프로모션 텍스트/설명/키워드)
- **개인정보 처리방침 URL**: `https://maesuscouter.kr/privacy.html` — 제출 전 브라우저에서 열리는지 확인
- **지원 URL**: 같은 도메인 또는 `mailto:qpt815@gmail.com` 가 안 되므로 블로그/페이지 주소 사용
- **App Privacy(앱 개인정보)**: AdMob 광고가 있으므로 "수집 안 함"이 **아닙니다**. 다음과 같이 신고:
  - 수집 항목: **식별자(기기 ID)**, **사용 데이터(광고 상호작용)**, **진단(크래시·성능)**
  - 각 항목 용도: **타사 광고(Third-Party Advertising)** / 사용자 신원과 **연결 안 함** / **추적(Tracking): 아니요** (앱이 ATT 추적 허용 팝업을 쓰지 않으므로)
  - 참고: Google 공식 "Apple 앱 개인정보 보호 세부정보" 문서( https://support.google.com/admob/answer/10262459 )
- **연령 등급**: 설문에서 전부 "없음" → 4+ (무제한 웹 접속 아님, 도박 아님)
- **가격**: 무료

**심사 노트(검토 정보)에 꼭 쓸 것** (그대로 복사해도 됨):

```
이 앱은 한국/미국 주식의 공개 데이터를 점수화해 보여주는 정보 앱입니다.
- 로그인이 없으므로 데모 계정이 필요하지 않습니다.
- 투자 자문이 아니며, 앱 내에 "투자 판단의 참고 자료일 뿐"이라는 면책 문구가 포함되어 있습니다.
- 데이터는 GitHub Pages에 매일 갱신되는 정적 JSON을 사용합니다.
- 수익화는 Google AdMob 배너/전면 광고입니다 (추적 팝업 없음).
```

제출 → 심사는 보통 1~3일. 통과되면 App Store 공개 🎉

## 나중에 구독을 붙일 때 (참고)

1. App Store Connect > 앱 > **구독** — 월간/연간 상품 생성 (Android와 같은 가격: ₩4,900/월, ₩39,000/년)
2. RevenueCat 대시보드 > 프로젝트에 **App Store 앱 추가** → In-App Purchase Key 업로드 → `appl_` API 키 복사
3. `www/premium.js`: `RC_API_KEY_IOS`에 `appl_` 키 입력, `FREE_MODE: false`
4. RevenueCat Entitlement `premium`에 iOS 상품 연결 (기존 Android 설정과 같은 Entitlement)
5. 페이월 심사 요건은 이미 충족: 구매 복원 버튼 ✓, 약관/개인정보 링크 ✓, 가격은 스토어 실제 가격으로 자동 표시 ✓
6. 다시 6단계(Run workflow) → 새 버전 제출

## 문제 해결

| 증상 | 원인/해결 |
|---|---|
| 업로드 단계 `No suitable application records were found` | 4단계(앱 생성)를 안 했거나 번들 ID 불일치 |
| `Authentication credentials are missing or invalid` | Secrets 4개 중 오타. APPSTORE_P8에 BEGIN/END 줄 포함 전체가 들어갔는지 확인 |
| 인증서 관련 실패 (`No signing certificate`) | API 키 권한이 App Manager 미만이면 발생 → 2단계에서 권한 확인 |
| TestFlight에 빌드가 안 보임 | 처리 중(최대 1시간). 이메일로 "Missing Compliance" 오면 → Info.plist에 면제 선언이 있으므로 정상적으로는 안 옴 |
| 심사 거절 4.2 (최소 기능) | 웹뷰 앱이라는 이유. 대응: 점수 산정·차트·정렬 등 앱 고유 기능 설명 + 오프라인 캐시 언급하여 재심사 요청 |
| 앱 실행 직후 크래시 (광고 관련) | `Info.plist` 의 `GADApplicationIdentifier` 가 삭제됐거나 형식 오류 (`~` 포함 App ID 여야 함) |
| 광고가 안 나옴 | TEST_MODE 확인 → 실제 ID면 AdMob 신규 앱 검토 대기(수일) 또는 앱스토어 연결 전 게재 제한 상태 |
| 이 PC에서 `npx cap ...` 실행하면 크래시 | node가 한글 경로에서 죽는 버그. `E:\Projects\_ioswork`(ASCII 경로 복사본)에서 실행할 것 — 아래 참고 |

### 로컬에서 iOS 프로젝트를 다시 만져야 할 때 (한글 경로 버그 우회)

`E:\Projects\_ioswork` 폴더가 ASCII 경로 작업용 복사본입니다 (이미 설치 완료 상태).
아이콘을 바꾸는 등 재생성이 필요하면:

```powershell
robocopy e:\Projects\앱만들기\capacitor-app E:\Projects\_ioswork /E /XD node_modules 출시 ios android
cd E:\Projects\_ioswork
npm ci
npx capacitor-assets generate --ios   # assets/ 의 icon-only.png, splash.png 로부터 재생성
robocopy E:\Projects\_ioswork\ios e:\Projects\앱만들기\capacitor-app\ios /MIR
```

## 유지보수 요약

- **웹 수정 후 새 iOS 빌드**: 커밋·푸시 → Actions에서 Run workflow (버전 그대로, 빌드 번호만 자동 ↑)
- **마케팅 버전 올리기** (예: 3.2): `capacitor-app/ios/App/App.xcodeproj/project.pbxproj` 안의 `MARKETING_VERSION = 3.1;` 두 곳 수정 → 푸시 → Run workflow → App Store Connect에서 새 버전 만들고 제출 (Android 쪽은 `scripts/patch-android.js` 의 VERSION_NAME과 맞추면 됨)
- **Android와 공통**: `www/` 는 두 플랫폼이 그대로 공유. `premium.js` 설정만 플랫폼별 키가 다름
